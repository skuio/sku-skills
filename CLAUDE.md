# CLAUDE.md — sku-skills

Project instructions for Claude Code working in this repo. See [`AGENTS.md`](./AGENTS.md) for the
full agent protocol and [`SKILL_SPEC.md`](./SKILL_SPEC.md) for the canonical format.

## What this is

An open-source library of e-commerce agent skills for the [SKU.io](https://developer.sku.io) API.
Skills are authored **once** in a canonical format (`skills/<domain>/<name>/`) and compiled to
Claude / OpenAI / Gemini outputs under `dist/`.

## Non-negotiables

- **Source of truth is `skills/` and `shared/`.** `dist/` is generated by `npm run build` and is
  git-ignored — never edit or commit it.
- **Real endpoints only.** Every path/method/field in a `skill.yaml` must come from the live API
  (<https://developer.sku.io>). Do not invent or guess endpoints. When in doubt, verify or leave out.
- **Least-privilege scopes.** `auth.scopes` = exactly what the operations require.
- **Guardrails required** for any create/update/delete/adjust skill (idempotency, confirm-before-send,
  don't-invent-ids).
- **`npm run check` must pass** before finishing any change.
- **Never commit secrets** — examples use placeholders (`$SKU_PAT`, `$SKU_TENANT`, fake ids).

## Workflow

```bash
npm ci
# edit skills/<domain>/<name>/{skill.yaml,INSTRUCTIONS.md}
npm run check      # validate schema + build all three targets
```

Match the structure and quality of the exemplars: `skills/products/find-product/` (simple),
`skills/orders/create-sales-order/` (request body + guardrails).

## Layout

| Path | Purpose |
| --- | --- |
| `skills/` | Canonical skills — **edit these** |
| `shared/` | Cross-cutting reference (auth, api overview, pagination, errors) |
| `schemas/skill.schema.json` | The `skill.yaml` schema (validated by `tools/validate.mjs`) |
| `tools/` | `build.mjs` (canonical → 3 targets), `validate.mjs`, `lib/` |
| `dist/` | Generated outputs — do not edit or commit |

## Do NOT

- Weaken `tools/validate.mjs` or the schema to make a skill pass — fix the skill.
- Commit `dist/`, `node_modules/`, or unrelated lockfile churn.
- Add real credentials or tenant data anywhere.
